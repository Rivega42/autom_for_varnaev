Интеграция с АУРА

Переключатель режима. Выкл — контур работает автономно, разъёмы /integration/* отвечают 501. Вкл — разъёмы открыты для АУРА. Применяется сразу, без перезапуска сервиса.

Камеры

    Камер нет. Заведите камеру в форме ниже или через сид-конфиг (db/seeds/object.yaml + scripts/seed.py --apply).

    Добавить камеру

    Имя должно совпадать с именем потока в go2rtc.yaml.

    Функции аналитики:

    ROI-зоны

    Клик по кадру — добавить вершину полигона (≥3). Зоны можно рисовать и прямо на живом видео — кнопка «Зоны ✎» в «Живом анализе».

    idтипвершин

    События (онлайн)

    Лента событий аналитики по помещению камеры; обновляется автоматически.

    времятипсообщение

    Живой анализ (скелет/распознавание)

    Браузерный анализ камеры: скелет, распознавание протирки, % покрытия зон, журнал и стоп-кадры — на том же ядре, что и сервер. События пишутся в журнал (Grafana).

    Камера не выбрана

    Выберите камеру в списке слева — откроются настройки аналитики, ROI-зоны, события и живой анализ.

    Пороги датчиков

    Критерии событий: превышение/возврат к норме. Применяются воркером в течение минуты.

    помещениеметрикаусл.знач.важность

    Правила уборки (санитарный контроль)

    Зона должна убираться не реже интервала; покрытие последней уборки — не ниже порога. Нарушение — событие «не убрано вовремя» (журнал/Grafana, уведомления).

    помещениезонаинтервал, чмин. %подписьвкл

    Контроль присутствия (рабочие зоны)

    В окне времени присутствие в рабочей зоне помещения должно фиксироваться не реже допустимого перерыва. Нарушение — событие «нет присутствия» (журнал/Grafana, уведомления). Времена окна — в поясе PRESENCE_TZ планировщика.

    помещениеокноперерыв, минвкл

    Расписания видеоанализа (таймер)

    Периодический запуск анализа. Изменения подхватываются планировщиком на следующем тике.

    имяисточникпомещениеинтервал, мин

    Отчёт за период (санинспекция/ППК)

    Уборки, просрочки и холодовая цепь (мин/макс/среднее, время вне нормы). CSV открывается в Excel.
