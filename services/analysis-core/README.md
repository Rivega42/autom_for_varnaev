# analysis-core — единое ядро видеоаналитики

Это **общий** модуль распознавания поз и действий, который используется
**одинаково** в двух местах:

- **в браузере** — живой просмотр в настройках камеры (`api-gateway`,
  страница `live.html`);
- **на сервере (Node)** — анализ по расписанию, по RTSP-потоку и по
  видеофайлу/пинку из API (воркер `video-analytics`).

Цель — убрать расхождение «браузер видит одно, сервер другое». Логика
перенесена **дословно** из рабочего браузерного PoC
`services/video-analytics/reference/motion-log.html` (эталон поведения —
`docs/07_VIDEO_ANALYTICS.md`).

## Что внутри

Модуль не зависит ни от DOM, ни от Node-API — это чистый ES-модуль.

- **Чистые примитивы** (дословно из PoC): `countReversals`, `motionShape`,
  `shapeText`, `angleAt`, `pip`, `range`/`mean`/`std`, `inBand`, `clamp01`,
  `r4`, `packLM`, `dist`, `vis`, `handPoint`. Это математика анализа без
  состояния.
- **`HeatGrid`** — серверный аналог heat-канваса: накопитель «протёртости»
  на сетке в landmark-координатах. Даёт процент покрытия зоны
  (`coverage(pts)`) тем же способом, что попиксельный `coverageByZone()` в
  PoC, но без `<canvas>`.
- **`AnalysisEngine`** — стейт-машина разбора кадра. На вход — массив
  landmark-точек MediaPipe Pose (нормализованные 0..1) и опциональные
  world-точки (3D). На выход (`analyze(lm, world, now)`) — список **событий**:

  ```js
  { text, color, isAct, snapshot, coverage? }
  ```

  где `text` — человекочитаемое сообщение (рус.), `isAct` — это «активность»
  (старт/конец уборки, действие), `snapshot: true` — PoC просил стоп-кадр
  (хост сам его делает), `coverage` — для финального события зоны
  (`{ zoneType, zoneName, pct }`).

## Граница ответственности

Ядро считает **что произошло** и **процент покрытия**. Отрисовка скелета,
визуальная заливка теплокарты и сам JPEG стоп-кадра — забота **хоста**
(браузер рисует на `<canvas>`, Node — через свой адаптер). Координаты внутри
ядра всегда landmark; зеркалирование (MIRROR) — чисто визуальное, ядро его не
учитывает (точки кистей и полигоны зон лежат в одной системе координат).

## Тесты

```bash
cd services/analysis-core
node --test
```

Покрыты чистые примитивы и дымовой сценарий движка (синтетическая «протирка
стола» → событие старта + покрытие зоны в финале).
