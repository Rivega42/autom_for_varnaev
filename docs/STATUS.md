# Что уже работает (статус v1)

> Снимок состояния ветки `main` на 2026-06-10. Перечислено только то, что
> **реально реализовано, слито в `main` и покрыто тестами** (336 тестов pytest +
> 22 Node-теста зелёные, `ruff`/`mypy` чисты, CI зелёный). Граница v1/v2 — в §13.

---

## 1. Оркестрация и развёртывание
- **`docker compose`**: весь контур одной командой; профили `release` (закрытые
  Nuitka-сборки), `seed` (массовый сидинг), `with-video`.
- **Сетевая сегментация**: `internal` (реальная изоляция, `internal: true`),
  `edge` (публикуемые порты и доступ к LAN объекта), `integration` (мост к АУРА,
  в v1 без обмена). Наружу опубликованы только api-gateway, Grafana и MQTT.
- **Тома**: `db_data`, `artifacts`, `mqtt_data`, `grafana_data` (данные переживают
  пересоздание контейнеров).
- **Миграции Alembic** применяются автоматически one-shot сервисом `migrate` до
  старта прикладных сервисов; те ждут готовности схемы и здоровья БД.
- **Healthchecks** у БД, log-service, api-gateway; правильные `depends_on`
  (воркер ждёт `log-service: healthy` — первые события не теряются).
- **Демо-режим без железа** (`docker-compose.demo.yml` + `bootstrap.sh --demo`):
  синтетические датчики → события → Grafana, без камер и узлов.
- **Скрипты**: `bootstrap.sh`, `fetch_web_assets.sh` (вендоринг MediaPipe/модели),
  `fetch_model.sh`, `init_env.py`, `seed.py`.

## 2. База данных
- **TimescaleDB** (PostgreSQL 16 + timescaledb).
- Таблицы: `rooms`, `sensor_nodes`, `cameras`, `camera_zones`, `sensor_readings`
  (гипертаблица), `events`, `artifacts`, `analysis_tasks`, `thresholds`,
  `schedules`. Read-only пользователь для Grafana.

## 3. Датчики (приём и события)
- **`ingest-sensors`**: подписка на MQTT, разбор показаний, запись в БД.
- **Пороги**: события `threshold_exceeded` / `back_to_normal` по правилам с
  гистерезисом; пороги настраиваются (общие и по помещению).
- **Контроль «тишины»** узла: событие `sensor_silent`, если узел молчит дольше
  порога; считается по **серверным** часам (сдвиг часов узла не даёт ложных
  тревог), однократно на эпизод.
- **`demo-sensors`**: генератор синтетических показаний для демо.
- **Прошивка ESPHome** (эталон `firmware/esphome/node.example.yaml`): ESP32-C3 +
  SHT41/SHT40 (воздух), MLX90614 (ИК-поверхность), LTR390 (УФ-индекс),
  GUVC-S10GD (УФ-C, аналог через ADC); метрики `uv_index`/`uv_c`.
- **MQTT-контракт** зафиксирован (`docs/08`): `monitoring/<node>/<metric>`.

## 4. Единый журнал событий
- **`log-service`**: приём событий по REST, чтение с фильтрами (время/тип/
  помещение), пагинация с ограничением `limit/offset`. Хранит `artifact_id`.

## 5. API-шлюз (единственный внешний REST-вход)
- Единый **конверт ответа** (`status/data/error/ts`), коды ошибок, OpenAPI
  (`/openapi.json`, `/docs`) — машинная версия контракта.
- **Аутентификация** по `X-API-Key` (сравнение за постоянное время); медиа-
  эндпойнты принимают ключ и в `?api_key=` (для `<img>`).
- **Рабочие эндпойнты**: `health`, `events` (+`/{id}`), `analytics-events`
  (действия и `coverage_report` из браузера, + сохранение стоп-кадра),
  `artifacts/{id}` (отдача файла), `analysis-tasks` (создать/список/статус),
  `readings`, `rooms`, `sensor-nodes`, `cameras` (+`/{id}`, PATCH),
  `cameras/{id}/snapshot`, `cameras/{id}/stream.mjpeg` (прокси go2rtc),
  `cameras/{id}/zones` (CRUD), `thresholds` (CRUD), `schedules` (CRUD).
- Пределы выборок (`limit`) и валидация дат (без зоны → UTC, иначе 422).
- **Разъёмы АУРА** `/integration/*` существуют и заглушены (`501 NOT_IMPLEMENTED`).

## 6. Веб-GUI (под `/ui`)
- Заведение и настройка **камер**, **помещений**, **узлов датчиков**.
- **ROI-зоны** камеры (рисование полигонов по кадру-превью), тумблеры функций
  аналитики, **пороги** и **расписания** — через интерфейс, без SQL.
- Кадр-превью и **живой MJPEG-поток** камеры.
- Лента событий по помещению камеры.

## 7. Видеоаналитика — единое ядро
- **`analysis-core`** — один движок (дословный порт рабочего PoC) для браузера
  **и** сервера: позы (руки/ноги/голова/корпус/присед/наклон), семейство уборки
  (стол/пол-швабра/подметание/окно) с гейтингом по ROI-зоне и одноручной
  протиркой, махи/хлопки/ходьба, падение/неподвижность/SOS, % покрытия зоны,
  сглаживание, чувствительность. Покрыт Node-тестами; браузерная копия байт-в-байт
  равна канону (страж в CI).

## 8. Видеоаналитика — браузер (живой анализ)
- **Нативно в карточке камеры** (без iframe): скелет-оверлей, распознавание,
  heat-заливка «протёртости» с обрезкой по зоне, журнал, **стоп-кадры**.
- События (действия и **отчёты о покрытии**) и стоп-кадры пишутся в журнал →
  видны в Grafana.
- MediaPipe и модель **вендорены локально** (клиент не ходит в интернет/CDN).

## 9. Видеоаналитика — сервер
- **Python-воркер** (`video-analytics`): очередь `analysis_tasks` (безопасный
  захват `FOR UPDATE SKIP LOCKED`), анализ RTSP-потока/видеофайла, детекторы
  поз/действий/«белого халата»/покрытия, события в журнал, **скриншот-артефакт**;
  лимит кадров для бесконечного RTSP.
- **Node-конвейер** (`video-analytics-node`) на том же ядре: события журнала в
  контрактной форме; **CLI воспроизведения записи** браузера на сервере
  (`bin/analyze.mjs`) — замыкает «записал в браузере → проверил на сервере».
- **Ротация артефактов**: скриншоты старше `ARTIFACTS_RETENTION_DAYS` (по
  умолчанию 30) удаляются автоматически (файл + строка в БД).
- **Планировщик** (`scheduler`): создаёт задания на анализ по расписаниям.

## 10. Grafana (дашборды как код)
- **Датчики** (ряды t°/влажность/ИК/УФ), **События** (журнал), **Видеоаналитика**
  (% покрытия зон, лента событий аналитики, **панель стоп-кадров**).
- Provisioning источника данных и алертов.

## 11. Качество и CI
- **Тесты**: 336 pytest + 22 Node-теста; **CI** гейтит `ruff` + `mypy` + `pytest`,
  Node-тесты ядра/конвейера, **валидацию `docker compose config`** и
  smoke-сборку образов всех сервисов.
- Конвенции: Conventional Commits, один issue → один PR.

## 12. Документация
- `docs/01..12` (архитектура, сеть, контракт API, модель данных, безопасность,
  план, видеоаналитика, MQTT, эксплуатация, развёртывание, быстрый старт на
  ноутбуке), `docs/customer/` (для заказчика), `docs/research/` (ESP32-CAM,
  Zigbee/mesh). Контракт-документы синхронны с кодом.
