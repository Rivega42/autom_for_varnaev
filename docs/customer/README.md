# Пакет документации для заказчика

Эта папка — **самодостаточный комплект, который можно передавать заказчику**. Она
не содержит внутренних материалов (процесс разработки, дизайн интеграции с АУРА,
схема БД и т.п.) — только то, что нужно и можно отдать для работы с системой.

Состав:

- [`РУКОВОДСТВО_ЗАКАЗЧИКА.md`](РУКОВОДСТВО_ЗАКАЗЧИКА.md) — единый документ:
  назначение, состав, развёртывание, заведение объекта, настройка видеоаналитики,
  работа с API, эксплуатация, FAQ и приложения (MQTT, эндпойнты, типы событий).
- [`diagrams/`](diagrams/) — схемы (SVG), встроенные в руководство: топология,
  сетевая модель, состав продукта, взаимодействие компонентов, жизненный цикл
  задания, поток события.

Для передачи заказчику достаточно отдать **только эту папку** (`docs/customer/`).

> Схемы в `diagrams/` — это копии рендеров из `docs/diagrams/`. При изменении
> исходных схем обновите копии здесь, чтобы пакет оставался актуальным.
