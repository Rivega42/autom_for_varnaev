# Пакет документации для заказчика

Эта папка — **самодостаточный комплект, который можно передавать заказчику**. Она
не содержит внутренних материалов (процесс разработки, дизайн интеграции с АУРА,
схема БД и т.п.) — только то, что нужно и можно отдать для работы с системой.

Состав:

- [`РУКОВОДСТВО_ЗАКАЗЧИКА.md`](РУКОВОДСТВО_ЗАКАЗЧИКА.md) — единый документ:
  назначение, состав, развёртывание, заведение объекта, настройка видеоаналитики,
  работа с API, эксплуатация, FAQ и приложения (MQTT, эндпойнты, типы событий).
- [`diagrams/`](diagrams/) — схемы (SVG), встроенные в руководство: топология,
  сетевая модель, состав продукта, взаимодействие компонентов, жизненный цикл
  задания, поток события.
- [`integration/`](integration/) — машиночитаемые артефакты для разработчиков
  АУРА: Postman-коллекция (`aura_integration.postman_collection.json`) и
  OpenAPI-фрагмент (`aura_integration.openapi.yaml`) интеграционной поверхности.

Для передачи заказчику достаточно отдать **только эту папку** (`docs/customer/`).

## Готовый документ (PDF / HTML)

Отрисованный документ — в [`build/`](build/):

- `build/Rukovodstvo_zakazchika.pdf` — PDF (схемы встроены, ссылки кликабельны);
- `build/Rukovodstvo_zakazchika.html` — самодостаточный HTML (открывается в любом
  браузере, схемы внутри).

Markdown-файл — это исходник; отрендеренные PDF/HTML собираются из него командой:

```bash
pip install markdown weasyprint
python scripts/build_customer_doc.py
```

После правки `РУКОВОДСТВО_ЗАКАЗЧИКА.md` или схем пересоберите документ этой командой.

> Схемы в `diagrams/` — это копии рендеров из `docs/diagrams/` (источник истины).
> Синхронизация автоматизирована: после изменения исходных схем выполните
> `python scripts/sync_customer_docs.py`. Рассинхрон ловит тест в CI
> (`scripts/sync_customer_docs.py --check`).
