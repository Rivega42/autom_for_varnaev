# config — конфиги для сервисов контура

Каталог монтируется в контейнер `scheduler` как `/config` (только чтение).

## Расписания планировщика

`scheduler` читает расписания из `/config/schedules.json` (переменная
`SCHEDULER_CONFIG`). Файл — JSON-массив записей `ScheduleEntry`. Если файла нет,
планировщик работает вхолостую (заданий не создаёт) — это не ошибка.

Чтобы включить расписания, скопируйте пример и отредактируйте:

```
cp config/schedules.example.json config/schedules.json
```

`schedules.json` в репозиторий не кладём (конфиг конкретного объекта) — он
попадает под `*.json` локально; в гите остаётся только пример.
